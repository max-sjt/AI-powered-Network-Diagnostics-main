# AI-assisted analysis - 弱网服务日志分析与向量库RAG系统

## 概述

本文件夹包含用于弱网服务日志截取、分析和**真正的向量库RAG（检索增强生成）**系统的核心组件。系统使用FAISS向量库存储网络知识库，通过相似度搜索检索相关知识，并结合AI模型进行智能分析，回答关于特定时间点网络状况的问题。

## 核心功能

### 🔍 日志截取与分析
- **实时日志捕获**：从弱网服务终端输出中提取网络指标
- **多指标解析**：支持RTT、TCP丢包率、流量、RSSI、网络质量等指标
- **时间点查询**：支持"xx时间点网络情况怎么样"的智能问答

### 🤖 真正的向量库RAG系统
- **FAISS向量库**：使用FAISS存储和检索网络知识库
- **本地嵌入算法**：实现简单的TF-IDF风格嵌入，不依赖外部API
- **相似度搜索**：基于向量相似度检索最相关的知识库信息
- **AI模型集成**：使用阿里百炼API进行智能分析
- **知识库检索**：从`network_knowledge_base.py`中检索相关知识
- **本地备用模式**：API不可用时自动切换到本地分析

## 快速开始

### 1. 配置API密钥
**⚠️ 重要：使用前必须配置您的阿里百炼API密钥**

在运行任何RAG服务之前，请先替换以下文件中的API密钥：
- `local_vector_rag_analyzer.py`
- `interactive_rag.py`
- `test_dashscope_api.py`
- 其他RAG系统文件

将 `YOUR_DASHSCOPE_API_KEY_HERE` 替换为您的真实API密钥。

### 2. 安装依赖
```bash
cd "/root/WEAK_NET/WEAK_NET/AI-assisted analysis"
chmod +x install_rag.sh
./install_rag.sh
```

### 3. 测试API连接
```bash
python3 test_dashscope_api.py
```

### 4. 运行向量库RAG分析服务
```bash
# 方法1：直接运行本地向量库RAG系统
python3 local_vector_rag_analyzer.py

# 方法2：交互式模式（推荐）
python3 interactive_rag.py

# 方法3：其他RAG系统
python3 vector_rag_analyzer.py      # 云端向量库RAG
python3 simple_rag_analyzer.py     # 简化版RAG
```

### 5. 使用日志截取工具
```bash
python3 log_capture.py
```

## RAG系统架构

### 🔧 本地向量库RAG系统（推荐）
- **向量存储**：使用FAISS存储网络知识库
- **嵌入算法**：本地TF-IDF风格嵌入，384维向量
- **检索方式**：相似度搜索，返回前4个最相关文档
- **优势**：不依赖外部API，响应快速，数据安全

### 🌐 云端向量库RAG系统
- **向量存储**：使用FAISS + 阿里百炼嵌入API
- **嵌入算法**：阿里百炼text-embedding-v1模型
- **检索方式**：高质量向量相似度搜索
- **优势**：嵌入质量更高，但需要网络连接

### 📝 简化版RAG系统
- **检索方式**：基于关键词的文本匹配
- **分析方式**：本地规则 + AI增强
- **优势**：轻量级，无外部依赖

## 向量库RAG工作流程

```
网络指标 → 构建查询 → 向量检索 → 相关知识 → AI分析 → 详细报告
    ↓           ↓         ↓         ↓        ↓        ↓
  解析日志    问题构建   相似度搜索  知识库   阿里百炼   分析报告
```

### 知识库检索示例
当分析RTT延迟问题时，系统会：
1. 将"RTT延迟分析"转换为向量
2. 在FAISS向量库中搜索相似文档
3. 检索到`rtt_analysis`、`quality_assessment`等相关知识
4. 将检索结果作为上下文提供给AI模型
5. 生成基于知识库的专业分析报告

## 分析报告内容

每次向量库RAG分析都会生成包含以下维度的详细报告：

1. **网络状况评估** - 基于知识库的综合评估
2. **问题识别** - 识别潜在的网络问题
3. **原因分析** - 基于知识库信息分析根本原因
4. **解决建议** - 引用知识库中的解决方案
5. **分析依据** - 说明使用了哪些知识库信息
6. **向量检索来源** - 显示检索到的相关文档

## 网络指标阈值

| 指标 | 优秀 | 良好 | 一般 | 较差 |
|------|------|------|------|------|
| RTT延迟 | <10ms | 10-30ms | 30-50ms | >50ms |
| TCP丢包率 | 0% | 0-0.5% | 0.5-1% | >1% |
| WiFi信号 | -30~-50dBm | -50~-60dBm | -60~-70dBm | <-70dBm |
| 质量评分 | 90+ | 80-90 | 70-80 | <70 |

## 技术特性

### 向量库技术
- **FAISS向量库**：高效的相似度搜索和聚类
- **本地嵌入算法**：TF-IDF风格嵌入，384维向量
- **文档分割**：800字符块，150字符重叠
- **向量存储**：本地持久化存储，支持增量更新

### API配置
- 使用阿里百炼API (`qwen-plus`模型)
- 支持`enable_thinking: false`参数避免Qwen3模型错误
- 自动重试机制（最多3次）
- 超时保护

### 解析能力
- 支持多种日志格式的正则表达式匹配
- 智能时间点匹配（支持最接近时间查找）
- 按接口分组指标分析
- 自动数据验证和清洗

### 分析算法
- **向量检索**：基于相似度搜索的知识库检索
- **多维度评估**：RTT、TCP丢包率、流量、RSSI、接口状态
- **知识库匹配**：自动匹配问题类型到相关知识
- **AI增强分析**：结合检索结果进行智能分析
- **本地备用模式**：API不可用时使用本地分析

## 故障排除

### API密钥配置问题
1. 确认已将所有文件中的`YOUR_DASHSCOPE_API_KEY_HERE`替换为真实API密钥
2. 检查API密钥格式是否正确（以`sk-`开头）
3. 确认API密钥有效且有足够的配额

### 向量库问题
1. 检查FAISS依赖是否正确安装
2. 确认向量存储文件权限
3. 重新构建向量库：删除`network_knowledge_vectorstore_local`文件夹

### API连接问题
1. 检查API密钥是否正确
2. 确认网络连接正常
3. 运行`test_dashscope_api.py`测试

### 日志解析问题
1. 确认日志格式符合预期
2. 检查时间戳格式是否为HH:MM:SS
3. 验证日志内容完整性

### 分析结果异常
1. 检查输入的时间点是否存在数据
2. 确认日志数据的时间范围
3. 使用`times`命令查看可用时间点
