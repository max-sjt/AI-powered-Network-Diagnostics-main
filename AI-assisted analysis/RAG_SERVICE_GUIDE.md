# RAG服务使用指南

## 概述

本指南详细介绍如何使用弱网服务日志分析与RAG系统。系统能够实时捕获弱网服务的日志输出，并通过AI模型进行智能分析，回答关于特定时间点网络状况的问题。

## 系统架构

```
弱网服务日志 → log_capture.py → optimized_network_rag.py → 阿里百炼API → 分析报告
                    ↓
            network_knowledge_base.py (知识库)
```

## 核心组件

### 1. 日志截取组件
- **文件**: `log_capture.py`
- **功能**: 实时捕获弱网服务终端输出
- **输出**: 结构化的网络指标日志

### 2. RAG分析服务
- **文件**: `optimized_network_rag.py`
- **功能**: 主要的RAG分析服务
- **特性**: 支持时间点查询和智能分析

### 3. 交互式服务
- **文件**: `interactive_rag.py`
- **功能**: 提供交互式命令行界面
- **特性**: 支持实时日志捕获和查询

### 4. 知识库
- **文件**: `network_knowledge_base.py`
- **功能**: 存储网络分析专业知识
- **内容**: 指标阈值、症状分析、故障排除建议

## 安装与配置

### 1. 环境要求
- Python 3.7+
- 稳定的网络连接（用于API调用）
- 阿里百炼API密钥

### 2. 安装依赖
```bash
cd "/root/WEAK_NET/WEAK_NET/AI-assisted analysis"
chmod +x install_rag.sh
./install_rag.sh
```

### 3. 配置API密钥
在代码中设置您的阿里百炼API密钥：
```python
api_key = "sk-your-api-key-here"
```

## 使用方法

### 方法1：直接运行RAG分析
```bash
python3 optimized_network_rag.py
```
这将运行示例分析，展示系统功能。

### 方法2：交互式模式
```bash
python3 interactive_rag.py
```

#### 交互式命令
| 命令 | 功能 | 示例 |
|------|------|------|
| `1` 或 `capture [duration]` | 启动日志捕获 | `1 300` |
| `2` 或 `stop` | 停止日志捕获 | `2` |
| `3` 或 `times` | 显示可用时间点 | `3` |
| `4` 或 `analyze [time]` | 分析特定时间点 | `4 00:13:24` |
| `5` 或 `help` | 显示帮助 | `5` |
| `6` 或 `quit` | 退出程序 | `6` |

### 方法3：命令行模式
```bash
# 分析特定时间点
python3 interactive_rag.py analyze 00:13:24

# 启动日志捕获
python3 interactive_rag.py capture 300
```

## 分析报告结构

每次分析都会生成包含以下6个维度的详细报告：

### 1. 整体网络健康状况
- 综合评估网络状态
- 整体性能评级
- 关键指标汇总

### 2. 各接口具体表现
- 详细分析每个网络接口
- 指标数值和状态评估
- 接口间对比分析

### 3. 发现的问题或异常
- 识别潜在问题
- 异常指标标记
- 问题严重程度评估

### 4. 可能的原因分析
- 分析问题根本原因
- 环境因素考虑
- 历史趋势分析

### 5. 改进建议
- 提供优化建议
- 具体操作步骤
- 预防措施

### 6. 是否需要进一步监控
- 监控建议
- 告警阈值设置
- 持续观察要点

## 网络指标阈值

### RTT延迟
- **优秀**: <10ms
- **良好**: 10-30ms
- **一般**: 30-50ms
- **较差**: >50ms

### TCP丢包率
- **优秀**: 0%
- **良好**: 0-0.5%
- **一般**: 0.5-1%
- **较差**: >1%

### WiFi信号强度
- **优秀**: -30~-50dBm
- **良好**: -50~-60dBm
- **一般**: -60~-70dBm
- **较差**: <-70dBm

### 质量评分
- **优秀**: 90+
- **良好**: 80-90
- **一般**: 70-80
- **较差**: <70

## 故障排除

### API连接问题
1. **检查API密钥**
   ```bash

   ```

2. **确认网络连接**
   - 检查网络连接是否正常
   - 确认防火墙设置
   - 验证DNS解析

3. **检查API配额**
   - 确认API密钥有效
   - 检查使用配额
   - 验证账户状态

### 日志解析问题
1. **确认日志格式**
   - 检查时间戳格式（HH:MM:SS）
   - 验证指标格式
   - 确认接口名称

2. **检查日志内容**
   - 确认日志完整性
   - 检查特殊字符
   - 验证数据范围

3. **调试解析过程**
   - 启用详细日志

   - 验证指标范围

2. **检查知识库**
   - 确认知识库完整性
   - 检查阈值设置
   - 验证分析逻辑

3. **检查API响应**
   - 确认API调用成功
   - 检查响应内容
   - 验证分析质量
