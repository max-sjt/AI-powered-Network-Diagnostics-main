# 网卡方面的知识

## Mellanox网卡：

现已被Nvidia收购。本质上是**智能网络处理器**

### 核心亮点包括：

- **高性能网络连接**：支持从10GbE到100GbE甚至更高的以太网速率，以及InfiniBand（IB）网络，提供极低延迟（如ConnectX-5延迟小于600纳秒）和高消息传输速率。
- **全面的协议卸载**：将TCP/IP、UDP等协议栈的处理工作从CPU卸载到网卡硬件，大幅降低CPU负载。
- **存储加速**：支持NVMe over Fabric（NVMe-oF）目标端卸载、纠删码（Reed-Solomon）计算卸载等，加速存储网络。
- **Overlay网络硬件卸载**：对VXLAN、NVGRE等隧道协议的封装和解封进行硬件处理，提升虚拟化网络的性能。

## PCIe虚拟化

### SR-IOV（Single Root I/O Virtualization）

允许一个PCIe设备在主机上呈现多个独立的虚拟设备

### PF（Physical Function物理功能）：

完整的PCIe功能用于管理网卡，如创建VF、配置QoS，控制权常在宿主机

### VF（Virtual Function虚拟功能）：

轻量级PCIe功能，由PF创建并分配资源。每个VF可以直通给一个虚拟机，让虚拟机直接”拥有“一块网卡硬件，实现近乎零损耗的IO性能

### SF（Sub-Function子功能）：

比VF更加轻量化，可以视为VF的进化版。SF不强制要求启用SR-IOV，可以与VF共存或独立与PF之上。同样拥有独立的收发队列和资源，适合再DPU（数据处理器）或者容器环境中为不同的服务提供隔离的网络功能。并且数量更多

## RDMA

核心思想是零拷贝、内核旁路、CPU卸载的数据传输

### 工作方式：

应用程序直接与网卡硬件交互，通过工作队列下发任务（WQE，Work Queue Element）

应用程序敲响门铃通知网卡有任务

网卡硬件直接DMA（直接内存访问）读写应用程序的内存，完成数据收发无需CPU参与

完成后，网卡将完成信息（CQE，Completion Queue Element）放入完成队列，应用程序通过轮询Polling获取结果，避免终端开销

### RDMA内存管理

应用程序先将内存注册为一个MR（Memory Region，内存区域），驱动会锁定这些内存页（防止被swap出去），并将虚拟地址到物理地址的映射关系告诉网卡硬件。同时PD（Protection Domain保护域）机制确保不同进程或租户的内存相互隔离，防止非法访问

## OVS-DPDK 软件定义网络的高性能组合

OVS（Open vSwitch）是流行的虚拟交换机，DPDK（Data Plane Development Kit

）提供用户态高性能数据包处理库

传统的OVS使用Linux内核的数据路径转发数据包，会受限于内核网络栈的性能瓶颈。而**OVS-DPDK将OVS的核心数据转发路径从内核空间转移到了用户空间**。

好处：

性能成倍提升：消除内核态与用户态之间频繁切换的开销

延迟更低：通过轮询和零拷贝等技术，数据包转发延迟显著降低，尤其适合电信金融等延迟敏感的业务

### 三级查找表（缓存）机制

**EMC（Exact Match Cache精确匹配缓存）**：最快。每个处理数据包的PMD线程有一个自己的小缓存，数据包进来后会先在这里尝试查找，匹配上就立即执行操作，但是存储的条目有限

**dpcls（datapath Classifier数据路径分类器）**：第二快。也称为Megaflow Cache。如果EMC没有找到，数据包就会来到这里，dpcls支持通配符匹配，比如只匹配目的IP，能容纳更多的流表项。为了优化查找效率，OVS会动态调整内部子表的查找顺序，把命中的子表排在前面

**ofproto (OpenFlow 控制器处理)**：最慢。如果前两级缓存都“查找失败”（Miss），意味着OVS不知道该对这个包做什么。这时，数据包会被通过“upcall”机制发送给用户空间的ofproto层，去询问OpenFlow控制器。这个处理路径速度比EMC慢10倍以上

**工作流程**是：数据包优先走最快路径（EMC），找不到就逐级向下，直到找到匹配的处理规则。一旦在dpcls或ofproto中找到了规则，这个规则就会被“学习”并安装到更快的缓存（如EMC）中，这样后续相同的“流”就能享受快速通道了。这种把数据包按“流”分组，然后批量处理（Batching）的方式，也是提升效率的关键。
